# Project Notes for Future Agents

This is a single monorepo for the full 12-week ADA Global Academy MicroMasters programme.

- Do not create separate repositories for individual weeks.
- Each week must have its own top-level folder: `week01-eda`, `week02`, ..., `week12`.
- Preserve reproducibility.
- Do not commit restricted, private, or unnecessarily large raw datasets.
- Add dataset access/reproduction instructions instead of committing restricted raw data.
- Keep assignment outputs, reports, figures, tables, and reproducible scripts.
- Use clear commit messages.
- Prefer feature branches for each week, such as `feature/week02-assignment`.
- Before merging to `main`, verify scripts run cleanly.
- Update the root README portfolio index after each week.
- Update `CHANGELOG.md` after each week.
- Maintain professional software engineering and research documentation standards.
